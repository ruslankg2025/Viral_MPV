Назначение проекта
Viral Monitor — SaaS-платформа для контент-команд, SMM-агентств и частных креаторов, которая закрывает полный цикл работы с коротким видео-контентом: мониторинг вирусных роликов в YouTube, Instagram, TikTok и VK, AI-разбор паттернов, генерация сценариев под профиль конкретного аккаунта, автоматическая сборка готового видео с AI-аватаром и публикация по расписанию. 
Решает три боли: ручной ресёрч и продакшн съедают 20+ часов в неделю, генерируемый контент без привязки к нише и аудитории конкретного канала даёт низкий отклик, а сборка и публикация требуют отдельного монтажёра и SMM-щика. Бизнес-кейс — заменить связку «аналитик + копирайтер + видеомонтажёр + SMM» одним инструментом с подпиской.

Функциональные модули
Мониторинг и сбор данных
- Парсеры YouTube (yt-dlp + YouTube Data API v3), Instagram (instaloader + Graph API для бизнес-аккаунтов), TikTok (TikTok Research API + fallback на HTML-скрейпинг через Playwright), VK (официальный API)
- Планировщик обходов на Celery Beat: периодичность от 15 минут до 24 часов на источник, приоритизация по тарифу пользователя
- Очередь задач на Redis + Celery с раздельными воркерами под CPU- и GPU-задачи (парсинг, транскрибация) и GPU-задачи (инференс локальной LLM)
- Дедупликация по хешу видео и нормализованным URL, инкрементальная догрузка метрик без полного перепарсинга
AI-анализ видео
- Мультипровайдерность: OpenAI (GPT-4o, GPT-4o-mini), Anthropic (Claude Sonnet 4.5, Haiku 4.5), Google (Gemini 2.5 Pro, Flash), локальный инференс через vLLM
- Единый абстрактный слой LLMProvider с автоматическим fallback по цепочке при 429/5xx/таймаутах и budget-aware роутингом (дешёвые модели для первичной классификации, дорогие — для глубокого разбора)
- Транскрибация через Whisper large-v3 (локально на GPU-сервере) + визуальный разбор ключевых кадров через vision-модели
- Кэширование результатов анализа в PostgreSQL с TTL и инвалидацией по изменению промпта или версии модели, sha256-ключ по (video_id, prompt_version, model)
### Настройка собственного аккаунта (центральный модуль)
Модуль-конфигуратор, который определяет поведение всех downstream-модулей (анализ, генерация, продакшн, публикация). Каждый подключённый аккаунт пользователя — это профиль с богатым контекстом, который инжектится в промпты LLM и в правила автоматизации.
- Идентификация и подключение: OAuth-привязка YouTube/Instagram/TikTok/VK, импорт исторических постов и метрик, автоматический первичный анализ существующего контента для извлечения tone of voice
- Профиль ниши и позиционирования: структурированные поля — основная ниша и подниши (из управляемого таксономии-справочника на ~200 категорий), УТП, ключевые темы, стоп-темы (что нельзя упоминать), конкуренты (ссылки на каналы для бенчмаркинга)
- Профиль аудитории: возрастные группы, география, язык, уровень дохода, болевые точки, уровень экспертности (новичок/средний/эксперт) — заполняется вручную или генерируется LLM из анализа комментариев и подписчиков
- Brand book: tone of voice (формальность 1–10, юмор 1–10, экспертность 1–10), запрещённая лексика, обязательные хэштеги и упоминания, шаблоны CTA, цветовая палитра и логотип для визуала
- Источники для мониторинга: список ссылок на каналы конкурентов и референсов, теги для группировки, частота обхода
- Кастомные промпты и ограничения для генерации сценариев:
  - Системный промпт уровня аккаунта (наследуется всеми сценариями)
  - Промпты-модификаторы для разных типов контента (Reels, Shorts, длинное видео, Stories)
  - Hard constraints: максимальная длительность, обязательная структура (хук → проблема → решение → CTA), запрет на конкретные форматы
  - Soft constraints: предпочитаемые форматы хуков, длина предложений, использование вопросов в начале
  - Версионирование всех настроек с возможностью отката и A/B-тестирования двух наборов промптов параллельно
- Параметры публикации: расписание выходов по дням недели и времени, часовой пояс, лимиты на постинг в сутки, правила автомодерации (требовать ручное подтверждение / публиковать автоматически)
Все настройки хранятся в JSONB-полях PostgreSQL с JSON Schema-валидацией, доступны через REST API и инжектятся в контекст LLM при каждом вызове генерации.
Генерация сценариев 
Текстовый модуль, который превращает извлечённые из мониторинга паттерны в готовые к продакшну сценарии под конкретный профиль аккаунта. Работает независимо от модуля продакшна — пользователь может остановиться на этапе текста и отдать его в свой существующий продакшн-пайплайн.
Промпт-инжиниринг на основе паттернов мониторинга + контекста профиля аккаунта (ниша, аудитория, brand book, кастомные промпты)
Версионирование шаблонов промптов в БД, A/B-тестирование на уровне аккаунта
Структурированный вывод (JSON Schema): хук, основной текст с таймкодами, CTA, описание визуальных сцен, suggested-хэштеги
Маршрутизация запросов между провайдерами с учётом BYOK-ключей и тарифных квот
История генераций с fork-ом и итеративной доработкой
Экспорт в Markdown, DOCX, JSON, прямая передача в AI Studio (для Business+)

### AI Studio — автоматизированный продакшн контента (Pro и выше)
Отдельный модуль с собственным GPU-пайплайном, который превращает текстовый сценарий в готовое к публикации видео без участия человека. Технологически и инфраструктурно вынесен в отдельную плоскость от генерации сценариев — другие очереди (rendering), другие воркеры, другая модель тарификации (минуты рендера + количество роликов).
- AI-аватар (talking head): интеграция с HeyGen API и D-ID как primary, self-hosted SadTalker / Hallo на GPU-сервере как fallback. Библиотека из 50+ пресетов или загрузка кастомного аватара (фото + образец голоса 30 секунд для клонирования)
- Синтез речи (TTS): ElevenLabs (основной), Coqui XTTS-v2 локально на GPU как fallback, поддержка SSML, автоматическая расстановка пауз и интонаций по таймкодам
- Подбор b-roll: автоматический поиск изображений и видео-фрагментов под каждую сцену через Unsplash API, Pexels API, Pixabay API, опционально Storyblocks. Релевантность через CLIP-эмбеддинги (open_clip_torch)
- AI-генерация визуала: для уникальных кадров — Flux.1, Stable Diffusion 3.5 локально на GPU либо облачный fal.ai. Промпты генерируются автоматически из описаний сцен
- Сборка видео: FFmpeg 7 + MoviePy в Celery-воркере, шаблоны монтажа (Reels 9:16, YouTube Shorts, классическое 16:9), автоматические субтитры из транскрипта TTS, наложение логотипа и брендинга из brand book
- Очередь рендера: отдельная Celery-очередь rendering с GPU-воркерами на dedicated node pool, приоритизация по тарифу, средний рендер 60-секундного Reels — 3–7 минут
- Превью и ручной контроль: перед публикацией пользователь смотрит готовое видео, заменяет аватар/голос/сцены, редактирует субтитры
### Автоматизация публикации (Pro и выше)
- Интеграции: YouTube Data API v3 (Shorts и обычные видео), Instagram Graph API (Reels, посты, Stories — только бизнес-аккаунты), TikTok Content Posting API, VK API (видео и клипы)
- Планировщик публикаций: календарь контента с drag-and-drop, привязка к расписанию из профиля аккаунта, автоматическое распределение сценариев по слотам
- Кросспостинг: одно видео одновременно в несколько платформ с автоадаптацией формата, индивидуальные описания и хэштеги через шаблоны
- Очередь и retry: Celery с exponential backoff, идемпотентность через external_id, dead letter queue
- Post-publish трекинг: через 1ч / 24ч / 7д автоматический сбор метрик (просмотры, лайки, сохранения, comments), привязка результатов к исходному сценарию для обучения промптов и A/B-выводов
- Модерация: опциональное ручное подтверждение перед публикацией, уведомления в Telegram/Email/webhook
Мультиаккаунтность и управление пользователями
- Multi-tenant архитектура: schema-per-tenant в PostgreSQL для изоляции данных команд Business-тарифа, row-level security для Free/Pro
- Ролевая модель: owner, admin, editor, viewer с гранулярными правами на блогеров, сценарии, биллинг
- OAuth 2.0 + email/password, TOTP 2FA, audit log всех действий в отдельной таблице с ретенцией 180 дней
- Изоляция API-ключей: пользователь может подключить собственные ключи OpenAI/Anthropic/Gemini (BYOK) — ключи шифруются AES-256-GCM, мастер-ключ в HashiCorp Vault
Трекинг собственных аккаунтов (Pro+)
- Подключение собственных YouTube Shorts/Instagram/TikTok/VK клипы/Pinterest клипы/ Дзен видео/ Одноклассники/  аккаунтов через OAuth, сбор метрик охватов, вовлечённости, роста подписчиков
- Сводный дашборд с графиками динамики (Recharts): просмотры по дням, engagement rate, топ-видео за период, сравнение с конкурентами из мониторинга
- Метрики считаются инкрементально и агрегируются в materialized views PostgreSQL с обновлением каждые 30 минут
- Алерты на аномалии (резкий рост/падение охватов) через email и webhook
Система тарифов и биллинг
- Stripe (основной, международные карты) + ЮKassa (РФ/СНГ), единый абстрактный слой PaymentProvider
- Подписочная модель с месячным/годовым циклом (скидка 20% на год), proration при апгрейде
- Счётчики использования в Redis с персистом в PostgreSQL раз в 5 минут: анализы видео, сгенерированные сценарии, количество отслеживаемых блогеров
- Rate-limiting через Redis (sliding window) с разными лимитами по тарифу, мягкий cap (предупреждение) и жёсткий (блокировка до следующего цикла)
- Webhook-и от платёжных систем обрабатываются идемпотентно через таблицу processed_webhook_events
Локальная LLM
- Отдельный GPU-сервер (NVIDIA A100 80GB или 2×RTX 4090), инференс через vLLM с continuous batching
- Модели: Llama 3.3 70B Instruct (основная для Free-тарифа и fallback), Qwen2.5 32B (для быстрых задач классификации)
- Очередь запросов через Redis Streams с приоритизацией: Business > Pro > Free, максимальное ожидание для Free — 120 секунд, иначе деградация до облачного провайдера за счёт платформы
- Мониторинг GPU через DCGM exporter + Prometheus, автоматический failover на облачные API при падении GPU-ноды
Структура UI
### Dashboard с трендами
Тренды ниши с фильтрами по платформам, метрики собственных аккаунтов (Pro+), сводка по мониторингу и квотам тарифа.
### Профиль аккаунта (центральный экран)
Многошаговый wizard для первичной настройки + детальный редактор. Вкладки: Идентификация и OAuth, Ниша и позиционирование, Аудитория, Brand book, Источники мониторинга, Промпты и ограничения генерации, Расписание публикаций. Превью того, как изменения повлияют на следующий сгенерированный сценарий.
### Мой контент и охваты (Pro+)
Динамика подписчиков, просмотров, engagement rate, таблица постов, сравнение с конкурентами.
### Управление блогерами
Список отслеживаемых аккаунтов с тегами, bulk-импорт, индивидуальные правила парсинга.
### Анализ видео
Транскрипт с таймкодами, структурный разбор, извлечённые паттерны, повторный анализ на другой модели.
### Редактор сценариев
Двухпанельный TipTap-редактор с версионированием, диффом, экспортом и кнопкой «Отправить в AI Studio» (активна для Pro+).
### AI Studio (Pro+)
Превью готового видео, выбор аватара/голоса/шаблона монтажа, замена сцен и b-roll, ручная правка субтитров, отправка в очередь публикации.
### Календарь публикаций (Pro+)
FullCalendar с drag-and-drop сценариев и готовых видео, статус (черновик / в очереди / опубликовано / ошибка), фильтрация по платформам.
### Настройки
BYOK-ключи провайдеров, OAuth-аккаунты соцсетей, команда (Business), 2FA, webhook-и, уведомления.
### Биллинг
Текущий тариф, прогресс-бары по счётчикам, история платежей, апгрейд с proration, счета и акты.
