# Монитор — инструкция пользователя

## Что такое Монитор

Раздел **Монитор** автоматически отслеживает Reels у выбранных Instagram-авторов
и подсвечивает ролики, у которых аномально высокая скорость набора просмотров
прямо сейчас — это «идеи в моменте». Не нужно каждый день обходить профили
руками: система сама собирает свежие ролики, считает скорость и ведёт shortlist
из топ-5 самых горячих рилсов на каждого автора.

YouTube и TikTok — кнопки в UI присутствуют, но интеграция в этом релизе
не активна. Все примеры ниже — про Instagram.

---

## Главный экран — анатомия

### Шапка
- **3 автора** — счётчик добавленных авторов (клик → модалка «Мои авторы»).
- **+ Добавить автора** — быстрый ввод @handle или URL профиля Instagram.

### Сегменты ленты
| Сегмент | Что показывает |
|---|---|
| **Для вас** | Алгоритмические рекомендации — заглушка, появится позже |
| **Мои авторы** | Только те ролики, что попали в watchlist (см. ниже) |
| **Все** | Все рилсы добавленных авторов за последние 48 часов |

### Платформы
`Все · IG · YT · TT · ❤` — фильтрует по соцсети. На текущем релизе работают только Instagram и «Все».

### Ниши
Бирюзовая пилюля «Все» + список ниш аккаунта + «Ещё ниши». Ниша определяется
**автоматически по хэштегам и ключевым словам в подписи ролика**, а не назначается
вручную на автора (один автор может рассыпаться по 3-м нишам).

### Поиск
По заголовку, описанию и @handle. Локальный фильтр поверх загруженной ленты.

### Фильтры времени публикации
`Все · 12ч · 24ч · 48ч · Неделя` — клиентский фильтр по `published_at`.
Не влияет на содержимое карточки, только на видимость. Реальный возраст ролика
всегда показан под просмотрами (`9 ч назад`).

### Сортировки
| Сортировка | Логика |
|---|---|
| **Горячее** | По скорости набора Δ24ч (для Мои авторы) или по velocity views/час (для Все) |
| **X-factor** | velocity × z-score — аномалия скорости относительно среднего по каналу |
| **Просмотры** | По абсолютному количеству просмотров |
| **Свежее** | По возрасту, от самого свежего |

Все фильтры и сортировки **сохраняются в localStorage** — после перезагрузки
страницы вернутся в том же виде.

## Карточка ролика

```
┌──────────────────────────┐
│  [превью]   ⭐ HIT       │  ← статус-бейдж в углу (только для «Мои авторы»)
│                          │
│  184 000                 │  ← просмотры
│  9 ч назад               │  ← реальный возраст
└──────────────────────────┘
  [ava] @demin_trader   Self-development
  ER 3.2%        18K/ч
  Δ24ч  +12 000 (+7%)
  в мониторинге ещё 2.6 дн
  [ Скопировать ссылку ]
```

| Метрика | Что значит |
|---|---|
| **184 000** (крупный белый) | Просмотры на момент последнего обхода |
| **9 ч назад** | Время с публикации ролика. Не зависит от фильтров |
| **⭐ HIT / ≈ STALLED / ↓ MISS** | Финальный статус ролика в watchlist (см. ниже) |
| **@demin_trader** + ✓ | Handle автора + бейдж verified, если профиль верифицирован |
| **Self-development** | Ниша ролика, определена автоматически по хэштегам |
| **ER 3.2%** | Engagement Rate: (лайки + комменты) ÷ просмотры |
| **18K/ч** (бирюзовый) | Velocity — сколько просмотров набирается в час сейчас |
| **18K/ч** (оранжевый) | Velocity ≥ 10 000/ч — «горячий» ролик |
| **Δ24ч +12 000 (+7%)** | Прирост просмотров за сутки. Зелёный = рост, оранжевый = падение. Показывается только если есть второй снапшот спустя ≥24 ч |
| **в мониторинге ещё 2.6 дн** | Сколько осталось до окончания TTL слежения. Скрывается у архивных |
| **Скопировать ссылку** | Копирует URL рилса в буфер обмена. Карточка целиком тоже кликабельна — открывает ролик в новой вкладке |

## Управление авторами — модалка «Мои авторы»

Каждая строка автора:

```
[avatar✓] @demin_trader        каждые [6 ч ▼]  видео [5]  [● тумблер]   [▶ Обновить]  [✕]
          обновл: 25 апр., 11:00 · 374K подп.
```

| Элемент | Действие |
|---|---|
| **Avatar** | Тянется через `/api/monitor/thumb/profile/by-handle/<h>` (proxy). Fallback — цветной кружок с инициалом |
| **✓ verified** | Маркер проверенного аккаунта Instagram |
| **обновл: …** | Время последнего успешного обхода |
| **374K подп.** | Followers count из Instagram Profile Scraper |
| **каждые 6 ч** | Интервал автоматического обхода (минимум 6 ч на текущем тарифе) |
| **видео [5]** | Сколько последних рилсов тянуть за один обход (override plan-лимита) |
| **● тумблер** | Активен / на паузе. При выключении — каскадно закрываются все active записи watchlist этого автора |
| **▶ Обновить** | Принудительный обход прямо сейчас, не дожидаясь cron. Параллельно обновляется профиль (cooldown 10 мин) |
| **✕** | Удалить автора и все связанные ролики/снапшоты (CASCADE) |

### Добавление
1. Нажать **+ Добавить автора**.
2. Вставить URL Instagram-профиля или `@handle`.
3. **Приватные профили блокируются** ошибкой `private_profile_not_supported (409)` — Apify не может тянуть такие.
4. После добавления нажать **▶ Обновить** — первый сбор пойдёт немедленно.

## Жизненный цикл «Мои авторы»

### 1. Сбор
Каждые 6 часов (или при ручном «▶ Обновить») система запрашивает у Apify последние 5 рилсов автора (`apify~instagram-reel-scraper`) и метрики профиля (`apify~instagram-profile-scraper`).

### 2. Ранжирование
Из всех рилсов **не старше 48 часов** и **старше 2 часов** (анти-шум: фильтрует ролики, у которых ещё не сформировалась velocity) выбирается **топ-5 по velocity** на каждого автора.

### 3. Добавление в watchlist
Выбранные ролики получают `status='active'` и отслеживаются 3 дня.
Дубли блокируются на уровне БД (partial unique index `WHERE status='active'`).

### 4. Финальный статус (по истечении TTL или раньше)

| Статус | Условие | Смысл |
|---|---|---|
| **active** | Только добавлен | Идёт мониторинг |
| ⭐ **hit** | Просмотры ×3 (≥+200%) **или** velocity ≥ 5 000/ч | Ролик «взлетел» — повтори формат |
| ↓ **miss** | TTL истёк, прирост < +20% | Не зашло |
| ≈ **stalled** | TTL истёк, прирост 20–200% | Небольшой рост, без взрыва |
| **closed** | Ручное «✕» на карточке или каскад при отключении автора | Снят с мониторинга вручную |

### 5. Cron автоматического отбора
Помимо ручных «Обновить», ежедневно в **08:00 UTC** запускается batch — отбирает топ-5 по всем активным источникам.

## Ежедневный сценарий

1. Открыть **Монитор → «Мои авторы»**.
2. Нажать **▶ Обновить** у нужного автора (или дождаться 08:00 UTC).
3. Отсортировать по **Горячее** — наверху самый сильный прирост за 24 ч.
4. Ролики с **velocity > 10K/ч** (оранжевый) и зелёным **Δ24ч** — кандидаты на разбор.
5. **⭐ HIT** — доказанный взлёт; кликнуть «Скопировать ссылку» → передать в Разбор.
6. **↓ MISS** — закрыть кнопкой «✕» если мешает.

## Лимиты тарифа (default `self`)

| Параметр | Значение | Что значит |
|---|---|---|
| `max_sources_total` | 50 | Не больше 50 авторов одновременно |
| `min_interval_min` | 360 | Минимум 6 часов между обходами одного автора |
| `max_results_limit` | 5 | Не больше 5 рилсов за один обход (× 3 = 15 на запрос Apify, потом фильтр) |
| `crawl_anchor_utc` | 00:00 | Базовая точка отсчёта расписания |

При попытке добавить 51-го автора → `409 plan_limit_reached`.
При выставлении интервала `< 360 мин` → молчаливый clamp до минимума.

## Частые вопросы

**Где «Δ24ч»?**
Появляется только когда есть второй снапшот старше 24 ч. Если ролик добавлен полчаса назад — нужно ещё одно «Обновить» через сутки.

**Почему 0 карточек после «Обновить»?**
- У автора нет рилсов за последние 48 часов
- Все его ролики моложе 2 часов (защита от шума)
- Профиль приватный (см. красное сообщение в строке автора)

**Почему карточки не появляются мгновенно?**
«Обновить» запускает Apify-actor, это 30–60 секунд. Скриншот reels + профиля идут параллельно, обновление в UI приходит после завершения.

**Можно ли добавить автора дважды?**
Нет, дубли по `(account_id, platform, handle)` блокируются. Удалённого можно добавить заново.

**Что если автор стал приватным?**
При следующем «Обновить» система это обнаружит, отключит автора (`is_active=false`) и каскадно закроет его watchlist-записи (`status='closed'`).

**Как сменить нишу ролика?**
Никак — ниша определяется автоматически по хэштегам и ключевым словам подписи. Если детектор ошибся — это баг, чините `_KEYWORD_NICHE` в `platforms/instagram.py`.

**Почему фильтр «12ч» оставил карточку «9 ч назад», а соседнюю «9 ч» — спрятал?**
Внутри хранится точное `hours_since_published` (например 8.7 и 11.4). Округление до часа в подписи — только для показа.
